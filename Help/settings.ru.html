Обнаружения
Модель Выберите модель, которую вы будете использовать для обнаружения.
  • Nocmig будет использовать для анализа модель, настроенную на ночные крики европейских птиц.
  • BirdNET будет использовать модель BirdNET разработанную Стефаном Калем и др. из Центра сохранения биоакустики К. Лизы Янг
Доверительный порог Этот параметр определяет минимальный уровень достоверности, необходимый модели для сообщения об обнаружении. Увеличение значения снижает количество ложных срабатываний, но увеличивает риск пропуска подлинных вызовов. Примечание Достоверность может быть скорректирована в любое время. Результаты обновляются даже после анализа.
Список

Местные птицы. Исключите птиц, которые вряд ли водятся в вашем регионе.

При выборе этого параметра появляется дополнительная панель. Пороговое значение представляет частоту проверки электронных птиц. Таким образом, значение 0,03 означает, что птицы, которые, по прогнозам, будут встречаться менее чем в 3% контрольных списков электронных птиц в этом районе, исключены. Если вы также установите флажок "Использовать неделю", это исключит птиц, которых не ожидалось в то время года, когда была сделана запись, которая считывается из временной метки файла.
Примечание В настоящее время ограничение для фильтров, зависящих от недели, заключается в том, что в сводке обнаружений будет использоваться список из последнего файла в пакете файлов. Например, если вы анализируете файлы за 22 апреля, 23 октября и 24 февраля в пакетном режиме, для сводки будет использоваться список за 24 февраля.

Ночные птицы. Это позволит исключить обнаружение пения птиц, которые обычно не издают звуков ночью. Если вы включите опцию "Использовать настройки местоположения" для модели BirdNET, список будет включать птиц из списка Местные птицы за исключением тех, которые не поют ночью.

Птицы. Включайте детекцию для всех птиц, за исключением классов, не относящихся к птицам.

Всё. Помимо всех видов птиц, показаны обнаружения объектов, не относящихся к птицам(включая сегменты, которые считались шумом).

Пользовательский. Предоставьте список для использования моделью. Пользовательские списки позволяют:

  1. Точно определите обнаружения, о которых вы хотели бы, чтобы Chirpity сообщала.
  2. Поиск только определенного вида.
Вы можете использовать разные пользовательские списки как для моделей BirdNET, так и для моделей Nocmig. Формат файла пользовательского списка - "научное название_общее название", где каждый вид выводится с новой строки. Важно отметить, что в пользовательском списке должны использоваться надписи на выбранном вами языке. Вы можете экспортировать начальный список в нужном формате из меню справки, используя Какие виды обнаружены?"
Анализ только в ночные периоды Ищите только ночные сигналы? Chirpity рассчитывает начало и конец гражданских сумерек по временным меткам ваших аудиофайлов с помощью заданного вами местоположения. Если ваша запись охватывает как дневное, так и ночное время, дневные периоды будут пропущены. Если вас интересуют только ночные сигналы, это дает два преимущества:
  1. Будет меньше ложных срабатываний, поскольку обе модели допускают больше ошибок, когда несколько видов вокализируют одновременно (например, во время утреннего пения)
  2. Анализ завершится быстрее, поскольку он игнорирует любой дневной период
Показывать статус вида в Красной книге IUCH в сводке Отобразите глобальный статус вида в Красном списке в сводной таблице по оценкеIUCN. Существует 7 категорий:
  1. Наименьшее беспокойство (Least Concern (LC))
  2. Под угрозой исчезновения (Near Threatened (NT))
  3. Уязвимый (Vulnerable (VU))
  4. Находящийся под угрозой исчезновения (Endangered (EN))
  5. Находящиеся под угрозой критического исчезновения (Critically ENdangered (CR))
  6. Вымерли в дикой природе (Extinct in the Wild)
  7. Исчезли (Extinct)
Если вы нажмете на значок, в вашем веб-браузере откроется страница оценки с описанием ареала вида и подробностями оценки. Данные взяты из: IUCN 2024. IUCN Red List of Threatened Species. Version 2024-2
Очистить кэш справочных вокализаций Метаданные сравнения видов, которые определяют, что вы видите, когда выбираете Сравнить с эталонными звуками в контекстном меню, сохраняются на диске. Если вы очищаете кэш, сохранённые данные удаляются, и при следующем запросе на сравнение данные о видах будут повторно запрошены с веб-сайта Xeno-Canto. Это может быть полезно, если эталонный файл удалён с сайта или если вы хотите обновить выбор звуков.
Местоположение по умолчанию Задайте местоположение по умолчанию для расчёта начала и конца гражданских сумерек, а также для создания списка Местных птиц. Этот параметр можно изменить для отдельных файлов, указав собственное местоположение в контекстном меню имени файла.
Система
Язык Выберите язык для меток Общее название, если существуют переводы. Переведенные метки были получены из eBird.
Бэкенд Эта опция доступна только при использовании модели Nocmig. Выберите способ расчёта прогнозов на вашем компьютере.
  • CPU (Процессор) будет использовать центральный процессор для прогнозирования, что обычно происходит быстрее, если у вас есть встроенная видеокарта.
  • WebGL будет использовать видеокарту. Если у вас есть выделенная видеокарта, это позволяет проводить анализ значительно быстрее.
  • WebGPU использует новую технологию и графические процессоры. Он похож на WebGL, но, в отличие от WebGL, ускоряет обработку на компьютерах Mac. WebGPU использует больше памяти, чем WebGL, и если значения количества потоков или размера пакета установлены слишком высокими, в Chirpity может закончиться память. В этом случае спектрограмма может исчезнуть с экрана или приложение может перестать отвечать на запросы. Используйте меньшее значение для количества потоков и/или размера пакета.
Потоки Этот параметр позволяет выбрать количество потоков, которые будут использоваться для прогнозирования. При использовании бэкенда процессора количество потоков автоматически настраивается в соответствии с количеством ядер процессора, доступных на компьютере. По умолчанию бэкенд графического процессора использует два потока. Настройка этого параметра, наряду с размером пакета, может увеличить скорость обработки.
Размер пакета Этот параметр определяет количество трехсекундных звуковых фрагментов, которые необходимо обработать для каждого запроса модели. Большие значения могут привести к ускорению обработки, особенно для длинных файлов, хотя слишком большое значение снизит скорость обработки и даже может вызвать проблемы с памятью. Оптимальное значение настроивается в зависимости от анализируемого формата файла.
Включить уведомления о завершении анализа Это полезно, если вы хотите запустить анализ Chirpity в фоновом режиме или в свернутом приложении. По завершении анализа вы получите системное уведомление.
Настройки звука
Регулировка громкости Если ваши записи очень тихие, вы можете увеличить громкость звука, повысив коэффициент усиления. Громкость звука можно увеличить на 50 децибел.
Нормализация громкости Если эта функция включена, уровни в аудиофайле будут нормализованы. Проще говоря, это позволит максимально увеличить диапазон громкости между самыми тихими и самыми громкими звуками. Этот эффект гарантирует, что все аудио воспроизводятся с одинаковой громкостью, а вокализации в тихих аудиофайлах будут легче слышны.
Фильтр высоких частот Этот фильтр удаляет звук ниже выбранной частоты. Предназначенный в первую очередь для записей, сделанных в городских условиях, он подавляет низкочастотный гул.
Фильтр низких частот (Low Shelf filter) Этот фильтр похож на фильтр высоких частот. Однако он не просто удаляет шум, а уменьшает громкость низкочастотного шума. Преимущество этого фильтра низких частот заключается в том, что вы можете уменьшить гул, но при этом слышать низкочастотные звуки. Вы можете установить пороговую частоту и силу воздействия фильтра (затухание)..
Экспорт аудио
Формат и битрейт Выберите формат аудиоданных, которые вы экспортируете из Chirpity: без потерь или с потерями.
Конвертировать в МОНО Объедините стереофонические треки в один канал. Это позволит уменьшить размер файла.
Оформление Эти два параметра можно использовать для:
  • Заполнение: Добавьте две секунды в начало и конец аудиозаписи, выбранной для экспорта
  • Затухание: примените эффект затухания к началу и концу экспортируемых файлов.
Аудио библиотека
Добавить в библиотеку при сохранении При сохранении набора результатов в архивной базе данных связанные с ним аудиофайлы будут преобразованы в формат библиотеки и сохранены в архиве.
Обрезать аудио из библиотеки до ночных периодов времени суток Это позволит обрезать начало или конец файла, чтобы удалить звук, записанный при дневной период суток. В частности, до окончания гражданских сумерек вечером и после окончания гражданских сумерек утром. Эти периоды устанавливаются в соответствии с настройками местоположения файла.
Расположение библиотеки SВыберите папку для хранения аудиотеки. Файлы будут организованы в соответствии со следующей структурой: 
        <Your chosen location> / Recording site / Year / Month / filename (Выбранное вами местоположение/Место записи/Год/Месяц/имя файла

Например: D:\Vögel\56075, Karthäuserhofgelände\2024\Juli\20240704_0130.flac
Формат библиотеки Библиотека может сохранять ваш звук в формате с потерями (lossy) или без потерь (lossless).
  • Формат с потерями: звук преобразуется в Ogg Vorbis. Это рекомендуемый вариант. Звук будет практически неотличим от оригинала в приложении — как для модели, так и для человеческого уха. Размер файла будет значительно уменьшен — настолько, что для 24 часов записи потребуется около 1 гигабайта памяти.
  • Формат без потерь: если выбрана опция без потерь, качество звука (битрейт, количество каналов и частота дискретизации) будет идентично исходному файлу в т.ч. для файлов WAV. При использовании формата FLAC размер архивных файлов будет примерно в два раза меньше, чем у этих же файлов в формате WAV.
Настройки спектрограммы
Цветовая карта Выберите цветовую тему для отображения спектрограммы или создайте свою собственную. Если вы выберете тему Пользовательская у вас будет возможность настроить цвета для пиковых, средних и тихих звуков в соответствии с вашими предпочтениями. Вы также можете настроить положение средней точки: при значении 0 или 1 спектрограмма будет двухцветной. Значения между этими двумя будут смешивать три цвета. Если вы установите средний цвет таким же, как один из других, вы сможете настроить контрастность спектрограммы с помощью настроек среднего положения.

В сочетании с настройками аудиофильтра пользовательская цветовая карта позволяет повысить контрастность/видимость вызовов с помощью выбранных вами цветов.
Оконная функция Для отображения спектрограммы доступны различные оконные функции. Каждая из них имеет отличающиеся характеристики, поэтому изменение оконной функции может улучшить внешний вид спктрограммы на дисплее.
Временная шкала Временная шкала спектрограммы имеет два режима:
  • Timecode отображает время, прошедшее с начала файла.
  • Время суток зависит от временных меток файла для определения фактического времени обнаруженных вокализаций. Если временная метка файла не отражает время записи, считывание будет неточным. Чтобы отредактировать время начала текущего файла, установите желаемую дату и время с помощью контекстного меню filename.
Расширенный
Отправка отфильтрованного звука для анализа По умолчанию настройки фильтров в разделе «Настройки звука» используются только для изменения звука в интерфейсе приложения. Если вы включите этот параметр, изменённый звук будет использоваться для анализа. Обычно это плохая идея, так как в большинстве случаев это снижает точность моделей. Однако, если в ваших записях много низкочастотного шума,возможно что изменение звука с помощью этих фильтров улучшит способность моделей к точному распознаванию.
Контекстный режим Этот режим доступен только для модели Nocmig. При включении модель будет использовать окружающий контекст (обстановку) при составлении прогнозов. Это помогает снизить вероятность ложноположительных срабатываний.
SNR фильтр Эта настройка также доступна только для модели Nocmig. Фильтр SNR (отношение сигнал/шум) может быть включен только при использовании бэкенга процессора. При включении Chirpity игнорирует звуковые сегменты без отчетливого звука вокализации. Для попытки прогнозирования требуется более сильный сигнал, когда установлены более высокие значения отношения сигнал/шум. Цель состоит в том, чтобы добиться значительного увеличения скорости, однако эта настройка может привести к тому, что Chirpity будет пропускать более тихие и удаленные вызовы.
Режим отладки Если вы столкнулись с проблемой при использовании Chirpity, вы можете включить режим отладки. После изменения этого параметра вам нужно будет выйти из приложения и перезапустить его, чтобы изменения вступили в силу. При запуске в режиме отладки откроются два окна, и каждое из них будет выводить отладочную информацию в консоль справа от окна. Этокрайне полезно, если вы хотите сообщить об ошибке, так как вы можете поделиться скриншотом любых появившихся ошибок (они будут отображаться красным в консоли).
