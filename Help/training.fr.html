Paramètres d'entraînement
Emplacement des sons d'entraînement Les fichiers audio utilisés pour l'entraînement doivent être organisés en dossiers, chacun contenant les sons d'une seule espèce. Le nom de chaque dossier doit suivre le format Nom scientifique_Nom commun, comme utilisé par les modèles BirdNET et Nocmig.
Emplacement du cache du jeu de données Le cache du jeu de données est l'endroit où Chirpity stocke les fichiers audio et les métadonnées traités. Si aucun emplacement n'est spécifié, celui des sons d'entraînement est utilisé. Il est recommandé de choisir un emplacement sur un SSD rapide pour des performances optimales.
Utiliser le cache existant s’il existe

La préparation du cache peut prendre un certain temps. Si cette option est cochée, le cache ne sera pas régénéré à chaque lancement de l'entraînement.

Si vous modifiez la répartition entraînement/validation, cette option sera réinitialisée.
Emplacement de sauvegarde du modèle

C’est ici que Chirpity enregistrera les fichiers du modèle entraîné.
Remplacer ou ajouter des étiquettes

Si « Remplacer les étiquettes » est sélectionné, le modèle enregistré contiendra uniquement les classes présentes dans les dossiers de votre jeu de données d'entraînement.

Si « Ajouter des étiquettes » est sélectionné, le modèle conservera les classes existantes de BirdNET et ajoutera toutes les nouvelles classes issues de vos dossiers d'entraînement.

Important : Les nouvelles classes ne doivent pas avoir le même nom qu'une classe existante. Pour remplacer une classe existante, ajoutez un "-" à la fin de son nom. Par exemple : Turdus iliacus_Mauvis-
Paramètres d'entraînement

Ces champs définissent les paramètres d'entraînement de base du modèle.

  • Époques : Nombre de fois que le jeu de données complet est passé à travers le modèle.
  • Taux d’apprentissage : Contrôle à quel point le modèle ajuste ses poids à chaque mise à jour.
  • Lissage des étiquettes : Technique pour rendre le modèle plus robuste en évitant la surconfiance dans une seule classe.
  • Utiliser la perte focale : Aide à gérer le déséquilibre des classes en accordant plus d’importance aux exemples difficiles.
  • Utiliser les pondérations de classe : Donne plus de poids aux classes sous-représentées pendant l'entraînement.
  • Utiliser un taux d’apprentissage décroissant : Réduit progressivement le taux d’apprentissage pour affiner les ajustements du modèle au fil du temps.
Jeu de données audio Ici, vous pouvez sélectionner une partie du jeu de données à utiliser pour la validation. Les données de validation ne sont pas utilisées pour entraîner le modèle, mais pour évaluer ses performances pendant l'entraînement.
Augmentations

Ces options contrôlent la façon dont les données d'entraînement sont modifiées pour améliorer la robustesse du modèle.

  • Mixup : Combine plusieurs exemples d'entraînement pour en créer un nouveau synthétique, aidant le modèle à mieux généraliser.
  • Décalage : Décale les données audio dans le temps pour créer de nouveaux exemples. Cela rend le modèle plus tolérant aux petits décalages temporels. Remarque : uniquement disponible avec le backend CPU
  • Ajouter du bruit de fond : Ajoute du bruit aux exemples d'entraînement, rendant le modèle plus robuste en environnement bruyant.
Classifieur

Ces paramètres définissent la structure du classifieur pour améliorer ses performances.

  • Unités cachées : Nombre de neurones dans la couche cachée du classifieur. Plus il y a d’unités, plus le modèle peut apprendre de motifs complexes, mais le risque de surapprentissage augmente. Si 0, aucune couche cachée n’est utilisée.
  • Abandon : Technique consistant à désactiver aléatoirement une partie des neurones pendant l'entraînement pour éviter le surapprentissage.
