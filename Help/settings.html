Predictions
Set Threshold Sets the confidence above which the model will report a detection. Higher values will reduce false positives, at the expense of missing genuine calls.
Model
  • DenseNet: Original model.
  • Efficient: New model. Faster, less memory intensive and more sensitive. More prone to false IDs when there is a lot of background noise.
Search for

Migrants and Owls. Just report detections relating to these species.

All birds. Include Local birds, including those not known to call at night.

Everything. This full list comprises 305 bird species and the following non-bird classes:

  • Ambient noise: wind, rain, pops and bangs. Things that go bump in the night...
  • Animals: cats, dogs, bats, etc.
  • Human: voices, whistles, etc.
  • Vehicles: Wheel and engine noise of cars, trucks, trains and aircraft. Also sirens and alarms (including domestic burglar alarms) and horns.

If you wish to see the full list of species Chirpity Nocmig was trained on, just explore the list offered when correcting a prediction.
Batch Size Sets the number of predictions to pass to the model each time. A higher number will make predictions up to 3x faster on long files. If set too high for your machine, the Chirpity model will crash as it runs out of memory. In that event, set a lower value, close and restart Chirpity.
Warm Up Because of the way the model works, the first prediction it is asked to do after Chirpity is opened will take much longer to process. When "Warm up" is checked, the model is passed a dummy prediction as soon as it is loaded. This happens in the background.
Spectrogram
Enable Whether to display the Spectrogram view.
Colour Displays the spectrogram in colour
Grayscale Displays the spectrogram in black and white.
Timeline Whether to display a timeline above the spectrogram. The timeline has two modes.
  • Time Elapsed displays the time from the beginning of the file.
  • Time of Day uses the file timestamp to calculate the actual time of the calls detected. If a file does not have a timestamp reflecting the time of the recording, this will give a spurious reading. The start time of the current file can be edited by selecting the appropriate date and time on the edit_calendar icon next to the file name on the transport controls panel.
