Predictions
Set Threshold Sets the confidence above which the model will report a detection. Higher values will reduce false positives, although that will increase the risk of missing genuine calls. A figure around 50% gives a good balance (i.e. it will detect most genuine calls while minimising false positive detections).
Model
  • DenseNet: Original model.
  • Efficient: New model. Faster, less memory intensive and more sensitive. More prone to false IDs when there is a lot of background noise.
Search for

Migrants and Owls. Just report detections relating to these species' calls (song detections are excluded).

All birds. Include birdsong as well as calls, also includes species not known to call at night.

Everything. This full list comprises 305 bird species and the following non-bird classes:

  • Ambient noise: wind, rain, pops and bangs. Things that go bump in the night...
  • Animals: cats, dogs, bats, etc.
  • Human: voices, whistles, etc.
  • Vehicles: Wheel and engine noise of cars, trucks, trains and aircraft. Also sirens and alarms (including domestic burglar alarms) and horns.

If you wish to see the full list of species Chirpity Nocmig was trained on, just explore the list offered when correcting a prediction.
Batch Size Sets the number of three second chunks of audio to pass to the model each time a prediction is requested. A higher number will make processing up to three times faster on long files. If set too high for your machine, the Chirpity model may crash as it runs out of memory. In that event, set a lower value, close and restart Chirpity.
Spectrogram
Colour Displays the spectrogram in colour
Grayscale Displays a monochrome spectrogram in shades of grey.
Timeline The spectrogram timeline has two modes:
  • Time Elapsed displays the time from the beginning of the file.
  • Time of Day uses files timestamps to calculate the actual time of the calls detected. If a file does not have a timestamp reflecting the time of the recording, this will give a spurious reading. The start time of the current file can be edited by selecting the appropriate date and time on the edit_calendar icon next to the file name on the transport controls panel.
