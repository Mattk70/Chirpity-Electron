Настройки обучения
Папка с аудиофайлами для обучения Аудиофайлы для обучения должны быть организованы по папкам, каждая из которых содержит записи одного вида. Названия папок должны быть в формате Научное_название_Обычное_название, как в моделях BirdNET и Nocmig.
Папка для кэша обучающего набора Здесь Chirpity будет хранить предварительно обработанные аудиофайлы и метаданные. Если путь не указан, будет использована папка с обучающими аудио. Для быстродействия рекомендуется использовать SSD.
Использовать существующий кэш, если доступен

Подготовка кэша может занять время. Если этот флажок установлен, Chirpity не будет пересоздавать кэш при каждом запуске обучения.

Если вы измените соотношение между обучающими и проверочными данными, эта опция будет сброшена.
Папка для сохранения модели

Путь, по которому Chirpity будет сохранять обученную модель.
Заменить или добавить метки

Если выбрано "Заменить метки", модель будет содержать только классы из ваших обучающих папок.

Если выбрано "Добавить метки", модель сохранит существующие классы BirdNET и добавит новые из ваших папок.

Важно: Новые классы не должны совпадать по названию с существующими. Чтобы заменить существующий класс, добавьте дефис в конце названия. Пример: Turdus iliacus_Дрозд-белобровик-
Параметры обучения

Эти поля задают основные параметры обучения модели.

  • Эпохи: Количество проходов всего набора данных через модель.
  • Темп обучения: Определяет, насколько сильно модель обновляется на каждом шаге.
  • Сглаживание меток: Предотвращает чрезмерную уверенность модели в одной категории.
  • Использовать фокальную функцию потерь: Помогает при несбалансированных классах, акцентируя внимание на сложных примерах.
  • Использовать веса классов: Учитывает дисбаланс классов при обучении.
  • Использовать уменьшение темпа обучения: Постепенно снижает скорость обучения для точной настройки.
Проверочная выборка Здесь вы можете выбрать долю аудиоданных для валидации. Проверочные данные не участвуют в обучении, но используются для оценки обобщающей способности модели в процессе обучения.
Аугментации

Эти опции управляют тем, как обрабатываются обучающие данные для повышения устойчивости модели.

  • Mixup: Объединяет несколько примеров в один синтетический для увеличения разнообразия.
  • Сдвиг во времени: Смещает сигнал во времени. Доступно только при использовании CPU-бэкенда
  • Фоновый шум: Добавляет шум к обучающим примерам для лучшей устойчивости в реальных условиях.
Классификатор

Эти параметры определяют структуру классификатора.

  • Скрытые нейроны: Количество нейронов в скрытых слоях. Большее число — более сложные закономерности, но выше риск переобучения. Если 0 — скрытых слоёв нет.
  • Dropout: Случайное отключение нейронов в обучении, помогает бороться с переобучением.
