検出
モデル 検出に使用するモデルを選択します。
  • Nocmig は、ヨーロッパの鳥の夜間飛行コールに調整されたモデルを使用して分析します。
  • BirdNET は、K. Lisa Yang Center for Conservation Bioacoustics の Stefan Kahl らによって開発された BirdNET モデルを使用します。
信頼度しきい値 この設定は、モデルが検出を報告するために必要な最小信頼度レベルを決定します。 値を上げると誤検出が減りますが、本物のコールを見逃すリスクが高まります。 注： 信頼度はいつでも調整できます。結果は分析後でも更新されます。
リスト

地元の鳥。あなたの場所で見つかる可能性の低い鳥を除外します。

このオプションを選択すると、追加のパネルが表示されます。しきい値 値は eBird チェックリストの頻度を表します。 したがって、0.03 の値は、その地域の eBird チェックリストの 3% 未満で予測される鳥を除外します。 「週を使用」をチェックすると、録音が行われた時期に予想される鳥を除外します - これはファイルのタイムスタンプから読み取られます。
注： 現在、週ごとのフィルターの制限は、検出のサマリーがバッチ内の最新のファイルのリストを使用することです。 たとえば、4月22日、10月23日、2月24日のファイルをバッチで分析する場合、サマリーには2月24日のリストが使用されます。

夜間のコール。これは、鳥の歌や夜間に通常鳴かない種のコールの検出を除外します。 BirdNET モデルで「位置設定を使用」を有効にすると、リストには 地元の鳥 リストの鳥が含まれ、夜間に鳴かない鳥は除外されます。

鳥。非鳥類のクラスを除外して、すべての鳥の検出を含めます。

すべて。すべての鳥の種に加えて、ノイズと見なされたセグメントを含む非鳥類の検出を表示します。

カスタム。モデルが使用するリストを提供します。カスタムリストを使用すると、次のことができます：

  1. Chirpity に報告してほしい検出を正確に定義する
  2. 特定の種のみを検索する
BirdNET と Nocmig モデルの両方に異なるカスタムリストを使用できます。カスタムリストファイルの形式は「学名_一般名」で、各種が新しい行にあります。重要なのは、カスタムリストが選択した言語のラベルを使用する必要があることです。 ヘルプメニューの「検出される種は何ですか？」リンクを使用して、正しい形式の開始リストをエクスポートできます。
夜間の期間のみを分析 夜間の検出のみを検索します。Chirpity は、オーディオファイルのタイムスタンプを使用して市民薄明の開始と終了を計算し、設定した位置を使用します。録音が昼間と夜間の両方の期間にまたがる場合、昼間の期間はスキップされます。夜間のコールにのみ関心がある場合、これには2つの利点があります：
  1. 両方のモデルは、多くの種が一度に鳴くとき（例：夜明けのコーラス）により多くの誤検出を行うため、誤検出が減ります。
  2. 昼間の期間を無視するため、分析が速く完了します。
サマリーに種の IUCN レッドリストステータスを表示 IUCN によって評価された種のグローバルレッドリストステータスをサマリーテーブルに表示します。7つのカテゴリがあります：
  1. 低懸念 (LC)
  2. 近危急 (NT)
  3. 危急 (VU)
  4. 絶滅危惧 (EN)
  5. 深刻な絶滅危惧 (CR)
  6. 野生絶滅
  7. 絶滅
アイコンをクリックすると、種の範囲と評価の詳細が表示される評価ページがウェブブラウザで開きます。データは次のソースから取得されます： IUCN 2024. IUCN Red List of Threatened Species. Version 2024-2
参照コールキャッシュをクリア コンテキストメニューで「参照コールと比較」を選択したときに表示される種の比較メタデータはディスクに保存されます。キャッシュをクリアすると、保存されたデータが削除され、次回比較を要求するときに Xeno-Canto サイトから種データが再要求されます。これは、参照ファイルがサイトから削除された場合や、コールの選択を更新したい場合に役立ちます。
デフォルトの場所 市民薄明の開始と終了を計算し、地元の鳥 リストを生成するために使用するデフォルトの場所を設定します。この設定は、ファイル名のコンテキストメニューでカスタム場所を割り当てることで個々のファイルに対して上書きできます。
システム
言語 翻訳が存在する場合、「一般名」ラベルの言語を選択します。翻訳されたラベルは eBird から取得されました。
バックエンド コンピュータで予測を計算する方法を選択します。
  • CPU は予測に CPU を使用します。統合グラフィックスカードを持っている場合、これが一般的に速いです。
  • WebGL はグラフィックスカードを使用します。専用グラフィックスカードを持っている場合、これが通常はるかに速いです。
  • WebGPU は新しい技術を使用し、グラフィックスプロセッサを使用します。WebGL に似ていますが、Mac での処理時間を短縮することが知られています。WebGPU は WebGL よりも多くのメモリを使用し、スレッドやバッチサイズの値が高すぎると Chirpity がメモリ不足になる可能性があります。この場合、スペクトログラムが画面から消えるか、アプリが応答しなくなる可能性があります。スレッドやバッチサイズの値を低く設定してください。
スレッド この設定では、予測に使用するスレッドの数を選択できます。 CPU バックエンドを使用する場合、スレッドの数はコンピュータで利用可能な CPU コアの数に自動的に設定されます。 デフォルトでは、GPU バックエンドは 2 つのスレッドを使用します。この設定を調整することで、バッチサイズとともに処理速度を向上させることができます。
バッチサイズ この設定は、各モデルリクエストに対して処理する 3 秒のオーディオチャンクの数を決定します。 値が大きいほど、特に長いファイルで処理速度が速くなる可能性がありますが、値が大きすぎると処理速度が低下し、メモリの問題が発生する可能性があります。 最適な値は、分析されるファイル形式によっても異なる場合があります。
分析完了通知を有効にする Chirpity 分析をバックグラウンドで実行したり、ウィンドウを最小化した状態で実行したい場合に便利です。分析が完了するとシステムアラートで通知されます。
オーディオ設定
音量調整 録音が非常に静かな場合、ゲインを上げることでオーディオの音量を増加させることができます。音量は最大 50 デシベルまで増加させることができます。
オーディオを正規化 有効にすると、オーディオファイルのレベルが正規化されます。簡単に言えば、最も静かな音と最も大きな音の間の音量範囲を最大化します。 この効果により、すべてのオーディオが同じ音量で再生され、静かなオーディオファイルのボーカリゼーションが聞き取りやすくなります。
ハイパスフィルター このフィルターは、選択した周波数以下の音を除去します。 主に都市環境での録音を対象としており、低周波のハム音を抑制します。
ロウシェルフフィルター このフィルターはハイパスフィルターに似ていますが、ノイズを単に除去するのではなく、低周波ノイズの音量を減少させます。 ロウシェルフフィルターの利点は、ハム音を減少させながら、低周波コールを持つ種を聞くことができることです。 しきい値周波数と適用するフィルターの強度（減衰）を設定できます。
オーディオエクスポート
フォーマットとビットレート Chirpity からエクスポートするオーディオクリップのために、ロスレスまたはロッシー形式を選択します。
モノラルに変換 ステレオトラックを単一のチャンネルに結合します。これにより、ファイルサイズが小さくなります。
デコレータ これらの2つのオプションを使用して：
  • パッド： エクスポートするオーディオの開始と終了に2秒を追加します
  • フェード： エクスポートするファイルの開始と終了にフェード効果を適用します。
オーディオライブラリ
保存時にライブラリに追加 結果をアーカイブデータベースに保存すると、関連するオーディオファイルがライブラリ形式に変換され、アーカイブ場所に保存されます。
ライブラリオーディオを夜間の期間にクロップ ファイルの開始または終了部分をクロップして、昼間に録音されたオーディオを削除します。具体的には、夕方の市民薄明の終了前と朝の市民薄明の終了後です。 これらの期間はファイルの位置設定に従って設定されます。
ライブラリの場所 オーディオライブラリを保存するフォルダーを選択します。ファイルは次の構造に従って整理されます： 
        <選択した場所> / 録音サイト / 年 / 月 / ファイル名

例： D:\Vögel\56075, Karthäuserhofgelände\2024\Juli\20240704_0130.flac
ライブラリ形式 ライブラリはロッシーまたはロスレス形式でオーディオを保存できます。
  • ロッシー： オーディオは Ogg Vorbis に変換されます。これは推奨オプションです。アプリケーション内でオーディオは元のものとほとんど区別がつかず、モデルと人間の耳の両方にとって同様です。 ファイルサイズは劇的に減少し、24時間の録音全体で約1ギガバイトのストレージが必要です。
  • ロスレス： ロスレスオプションを選択すると、オーディオの忠実度（ビ Chirpity Nocmig 設定

    スペクトログラムの設定
    カラーマップ スペクトログラム表示のカラーテーマを選択するか、自分で作成します。'カスタム'を選択すると、ピーク、ミッド、クワイエットサウンドの色を個人の好みに合わせて設定するオプションが表示されます。また、ミッドポイントの位置を調整することもできます：値が0または1の場合、スペクトログラムは2色になります。中間の値は3色をブレンドします。ミッドカラーを他の色のいずれかと同じに設定すると、ミッドポジションの調整を使用してスペクトログラムのコントラストを調整できます。

    オーディオフィルターの調整と組み合わせて、カスタムカラーマップを使用すると、選択した色を使用してコールのコントラスト/視認性を向上させることができます。
    ウィンドウ関数 スペクトログラム表示にはさまざまなウィンドウ関数が利用できます。それぞれにわずかに異なる特性があるため、ウィンドウ関数を変更すると、表示されるコールの外観も向上する可能性があります。
    タイムライン スペクトログラムのタイムラインには2つのモードがあります：
    • タイムコード ファイルの先頭からの経過時間を表示します。
    • 時刻 ファイルのタイムスタンプを使用して検出されたコールの実際の時間を決定します。 ファイルのタイムスタンプが録音時間を反映していない場合、読み取りが不正確になります。 現在のファイルの開始時間を編集するには、ファイル名のコンテキストメニューを使用して希望の日付と時刻を設定します。
    高度な設定
    フィルタリングされたオーディオを分析に送信 デフォルトでは、オーディオ設定のフィルター設定はアプリケーションインターフェース内のオーディオを変更するためにのみ使用されます。この設定を有効にすると、変更されたオーディオが分析に使用されます。これは通常、ほとんどの状況でモデルの精度を損なうため、良いアイデアではありません。ただし、録音に低周波ノイズがかなり含まれている場合、これらのフィルターを使用してオーディオを変更することで、モデルの正確な検出能力が向上する可能性があります。
    コンテキストモード このモードはNocmigモデルでのみ利用可能です。有効にすると、モデルは予測を行う際に周囲のコンテキストを使用します。これにより、誤検出を軽減するのに役立ちます。
    SNRフィルター この設定もNocmigモデルでのみ利用可能です。SNR（信号対雑音比）フィルターは、CPUバックエンドを使用している場合にのみ有効にできます。 有効にすると、Chirpityは明確な音イベントのないオーディオセグメントを無視します。 より高い信号対雑音値が設定されると、予測が試みられるためにはより強い信号が必要です。 目的は大幅な速度向上を提供することですが、この設定により、Chirpityがより静かで遠くのコールを見逃す可能性があります。
    デバッグモード Chirpityを使用している際に問題が発生した場合、デバッグモードを有効にできます。この設定を変更すると、変更を有効にするためにアプリケーションを終了して再起動する必要があります。デバッグモードで起動すると、2つのウィンドウが開き、それぞれのウィンドウの右側にあるコンソールにデバッグ情報が表示されます。 これはバグを報告したい場合に非常に役立ちます - エラーが表示されるコンソールのスクリーンショットを共有できるためです（これらはコンソールに赤で表示されます）。
