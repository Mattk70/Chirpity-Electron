Flux de travail suggéré

  1. Ouvrez un fichier audio, des fichiers ou un dossier d'audio. Il existe trois manières de procéder :
    1. Glissez-déposez les fichiers ou dossiers dans l'application Chirpity Nocmig
    2. Dans l'application Chirpity Nocmig, sélectionnez Fichier > Ouvrir un fichier audio ou appuyez sur Ctrl-O et sélectionnez les fichiers à ouvrir
    3. Cliquez droit sur les fichiers audio dans l'Explorateur Windows (ou Finder sur un Mac) et sélectionnez "Ouvrir avec" > Chirpity
    Idéalement, les fichiers doivent être ceux enregistrés par votre enregistreur numérique avec leur horodatage d'origine. Dans ce cas, vous pourrez profiter du mode Heure de la journée pour la chronologie du spectrogramme, qui affichera l'heure exacte à laquelle les cris ont été détectés. Sinon, vous pouvez toujours afficher (et enregistrer) l'heure correcte pour le fichier et les détections si vous modifiez son heure de début. Cela peut être modifié en cliquant droit sur le nom du fichier dans la barre de contrôle.
  2. Sélectionnez "Analyser le fichier actuel" dans le menu Analyse. Selon la longueur du fichier, cela peut prendre plusieurs minutes. Un indicateur de progression apparaîtra pour estimer le temps restant. Pour arrêter l'analyse à tout moment, appuyez sur la touche Échap.
Explorer les détections
  1. Lorsque le modèle effectue des détections, elles apparaissent dans un tableau sous le spectrogramme. En cliquant sur une ligne de détection dans le tableau, le curseur se positionnera au début de cette détection et mettra en évidence son emplacement sur le spectrogramme. En cliquant droit sur la ligne, un menu contextuel s'affiche. Ce menu contient des options spécifiques à la détection actuellement mise en surbrillance. Vous pouvez appuyer sur la barre d'espace pour lire à partir du début de la sélection et continuer la lecture jusqu'à ce que vous appuyiez à nouveau sur la barre d'espace. Cliquer sur le spectrogramme déplacera le curseur à cette position. Si la lecture est en cours, elle continuera, vous permettant ainsi de rejouer un cri plusieurs fois.
  2. Si plusieurs détections s'appliquent à une même période, un cercle sera affiché à côté du nom de l'espèce. En cliquant sur ce cercle, les autres espèces détectées dans la région s'afficheront, ainsi que leur niveau de confiance.
Résumé et filtrage des détections
  1. Une table récapitulative apparaît à gauche des résultats une fois que le modèle a terminé son analyse. Cette table permet de filtrer les résultats affichés dans le tableau principal. Cliquez sur le nom de l'espèce dans le résumé pour limiter l'affichage à cette espèce uniquement. Cela est extrêmement utile si plusieurs centaines des cris de Rouge-gorge ont été détectés !
Détections personnalisées
  1. Parfois, un cri est manqué ou une détection est erronée en raison de la position de la fenêtre d'analyse par rapport au cri. Vous pouvez déplacer la fenêtre en la faisant glisser, ou créer une nouvelle fenêtre en cliquant et en traçant une région personnalisée sur le spectrogramme. Une fois une nouvelle région créée, sélectionnez search Analyser dans le menu contextuel ou Analyse > Analyser l'intervalle sélectionné dans le menu. Les détections de la région sélectionnée apparaissent dans une fenêtre superposée et peuvent être enregistrées ou rejetées.
Modification des résultats
  1. Il est possible de supprimer des résultats en sélectionnant delete_forever Supprimer l'enregistrement dans le menu contextuel. Cette option peut être appliquée par détection dans le tableau des résultats, ou par espèce dans le résumé.
Enregistrement des résultats

  1. Note : Les résultats ne sont pas enregistrés par défaut.

    Enregistrez les résultats en sélectionnant Enregistrements > archive Enregistrer dans l'archive.
Restauration des résultats

Lorsque vous enregistrez vos détections dans Chirpity, une base de données est mise à jour avec les détails des détections. Si vous rechargez votre fichier ultérieurement et sélectionnez "Obtenir les résultats pour le fichier actuel", les résultats enregistrés seront restaurés. Note : Si le fichier original est manquant car il a été compressé dans l'un des formats pris en charge ('.mp3', '.wav', '.mpga', '.ogg', '.opus', '.flac', '.aac'), mais conserve le même nom avec une nouvelle extension, Chirpity trouvera et chargera ce fichier et utilisera l'heure d'origine.

Depuis le menu des enregistrements, vous pouvez également parcourir les détections sauvegardées par espèce et dans une plage de dates de votre choix. De plus, vous pouvez générer des graphiques des espèces détectées au fil du temps.

Exportation des résultats
Il existe plusieurs façons d'exporter les détections.
  1. Exportez un extrait audio d'une détection en sélectionnant music_note Exporter un extrait audio dans le menu contextuel. Par défaut, le nom du fichier contiendra le nom de l'espèce et l'horodatage de la détection. Le menu Paramètres vous permet de contrôler le format et la qualité des fichiers exportés, ainsi que d'ajouter une transition en fondu au début et à la fin de l'extrait audio.
  2. Exportez en 'lot' jusqu'à 1000 détections audio d'espèces en sélectionnant music_note Exporter des extraits audio dans le menu contextuel du tableau récapitulatif. Un dossier vous sera demandé pour enregistrer les fichiers.
  3. Exportez un fichier CSV des détections en sélectionnant ios_share Exporter les résultats au format CSV dans le menu Fichier. Les résultats seront exportés dans un fichier CSV dans le dossier choisi.
  4. Sauvegardez les détections au format étiquette pour Audacity. Sélectionnez Fichier > Enregistrer les étiquettes Audacity dans le menu. Par défaut, le fichier d'étiquette portera le même nom que le fichier audio, avec une extension ".txt".
