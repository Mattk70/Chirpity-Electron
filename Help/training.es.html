Las funciones de entrenamiento solo están disponibles cuando se utiliza el modelo BirdNET
Ubicación del Audio de Entrenamiento Los archivos de audio para el entrenamiento deben estar organizados en carpetas, y cada carpeta debe contener archivos de audio de una sola especie. Los nombres de las carpetas deben ser el nombre de la especie, en el formato Nombre científico_Nombre común utilizado por los modelos BirdNET y Nocmig.
Ubicación de la Caché del Conjunto de Datos La ubicación de la caché del conjunto de datos es donde Chirpity guarda los archivos de audio procesados y los metadatos. Si no se selecciona ningún valor, se utilizará la ubicación del audio de entrenamiento. Es recomendable elegir una ubicación en un SSD rápido para un rendimiento óptimo.
Usar la caché del conjunto de datos si existe

Preparar la caché del conjunto de datos lleva algo de tiempo. Cuando esta opción está seleccionada, la caché no se volverá a crear cada vez que se inicie el entrenamiento.

Si cambias la división entre entrenamiento y validación, esta opción se restablecerá.
Ubicación para Guardar el Modelo

La ubicación para guardar el modelo es donde Chirpity almacenará los archivos del modelo entrenado.
Reemplazar o Añadir Etiquetas

Si se selecciona "Reemplazar etiquetas", el modelo guardado tendrá las clases de las carpetas de tu conjunto de entrenamiento.

Si se selecciona "Añadir etiquetas", el modelo guardado conservará las clases existentes de BirdNET y añadirá cualquier clase nueva de las carpetas de tu conjunto de entrenamiento.

Importante: Las nuevas clases no pueden compartir el mismo nombre que una clase existente. Para sobrescribir una clase existente, debes añadir un "-" al nombre de la clase. Por ejemplo: Turdus iliacus_Zorzal alirrojo-
Parámetros de Entrenamiento

Estos campos configuran los parámetros básicos de entrenamiento del modelo.

  • Épocas: Número de veces que tu conjunto de datos pasa por el modelo durante el entrenamiento.
  • Tasa de Aprendizaje: Controla cuánto se ajusta el modelo en respuesta al error estimado cada vez que se actualizan los pesos.
  • Suavizado de Etiquetas: Técnica para hacer que el modelo sea más robusto evitando que se vuelva demasiado confiado en una sola clase.
  • Usar pérdida focal: Técnica para abordar el desequilibrio de clases enfocando más en ejemplos difíciles de clasificar.
  • Usar pesos por clase: Técnica para dar más importancia a las clases subrepresentadas durante el entrenamiento.
  • Usar decaimiento de la tasa de aprendizaje: Técnica para reducir la tasa de aprendizaje con el tiempo, permitiendo ajustes más finos conforme avanza el entrenamiento.
Conjunto de Datos de Audio Aquí puedes seleccionar una fracción del conjunto de datos para reservarla como validación. Los datos de validación no se usan para entrenar el modelo, sino para evaluar su rendimiento — su capacidad de generalizar — durante el entrenamiento.
Aumentaciones

Estas opciones controlan cómo se aumenta el conjunto de datos de entrenamiento para mejorar la robustez del modelo.

  • Mixup: Técnica que combina dos o más ejemplos de entrenamiento para crear un nuevo ejemplo sintético. Esto puede ayudar al modelo a generalizar mejor al exponerlo a una mayor variedad de entradas.
  • Desplazamiento (Roll): Técnica que desplaza los datos de audio a lo largo del eje del tiempo, creando nuevos ejemplos de entrenamiento al mover la señal de audio.
  • Mezclar con ruido de fondo: Técnica que añade ruido de fondo a los ejemplos de entrenamiento, haciendo que el modelo sea más robusto en entornos ruidosos. Para usar esta configuración, debes tener una carpeta llamada 'Background_Background', que contenga ruido de fondo, en la ubicación de tu audio de entrenamiento.
Clasificador

Estas opciones controlan cómo se estructura el clasificador para mejorar el rendimiento.

  • Unidades Ocultas: Se refiere al número de neuronas en las capas ocultas del clasificador. Aumentar este número permite al modelo aprender patrones más complejos, aunque también puede provocar sobreajuste. Si es 0, no se usan capas ocultas.
  • Dropout: Técnica que pone aleatoriamente a cero una fracción de las unidades de entrada en cada actualización durante el entrenamiento, lo que ayuda a prevenir el sobreajuste.
