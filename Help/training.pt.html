Local dos Áudios de Treinamento Os arquivos de áudio para treinamento devem estar organizados em pastas, com cada pasta contendo os áudios de uma única espécie. Os nomes das pastas devem seguir o formato Nome científico_Nome comum, usado pelos modelos BirdNET e Nocmig.
Local de Cache do Conjunto de Dados Este é o local onde o Chirpity armazena os arquivos de áudio processados e metadados. Se nenhum valor for definido, o local dos áudios de treinamento será usado. Recomenda-se uma localização em SSD rápido para melhor desempenho.
Usar cache existente se disponível

A preparação do cache pode levar algum tempo. Quando esta opção estiver marcada, o cache não será recriado toda vez que o treinamento começar.

Se você alterar a divisão entre treinamento e validação, esta opção será redefinida.
Local para Salvar o Modelo

Local onde o Chirpity irá armazenar os arquivos do modelo treinado.
Substituir ou Adicionar Rótulos

Se “Substituir rótulos” estiver selecionado, o modelo salvo terá apenas as classes das pastas do seu conjunto de dados de treinamento.

Se “Adicionar rótulos” estiver selecionado, o modelo manterá as classes existentes do BirdNET e adicionará quaisquer novas classes das suas pastas.

Importante: Novas classes não podem ter o mesmo nome de uma classe existente. Para substituir uma classe existente, adicione um “-” ao final do nome. Exemplo: Turdus iliacus_Pião-europeu-
Parâmetros de Treinamento

Esses campos definem os parâmetros básicos do treinamento do modelo.

  • Épocas: Número de vezes que o conjunto de dados passa pelo modelo durante o treinamento.
  • Taxa de Aprendizado: Controla o quanto o modelo é ajustado com base no erro estimado a cada atualização dos pesos.
  • Label Smoothing: Técnica para evitar que o modelo fique excessivamente confiante em uma única classe.
  • Usar perda focal: Técnica para lidar com desbalanceamento entre classes, dando mais peso a exemplos difíceis.
  • Usar pesos de classe: Dá mais importância a classes sub-representadas.
  • Usar decaimento da taxa de aprendizado: Diminui a taxa de aprendizado ao longo do tempo para ajustes mais finos.
Conjunto de Dados de Áudio Aqui você pode selecionar uma fração do conjunto de dados para usar como validação. Os dados de validação não são usados no treinamento, mas servem para avaliar o desempenho do modelo — sua capacidade de generalização — durante o treinamento.
Aumentações

Essas opções controlam como os dados de treinamento são aumentados para melhorar a robustez do modelo.

  • Mixup: Combina dois ou mais exemplos para criar um novo exemplo sintético, expondo o modelo a mais variações.
  • Rolagem: Desloca os dados de áudio no tempo, criando novos exemplos.
  • Adicionar ruído de fundo: Adiciona ruído aos exemplos de treinamento para tornar o modelo mais robusto a ambientes ruidosos. Para usar esta configuração, você deve ter uma pasta chamada 'Background_Background', contendo ruído de fundo, no local dos seus áudios de treino.
Classificador

Essas opções controlam a estrutura do classificador para melhorar o desempenho.

  • Unidades Ocultas: Número de neurónios nas camadas ocultas. Mais unidades permitem aprender padrões mais complexos, mas podem causar sobreajuste. Se for 0, não há camadas ocultas.
  • Dropout: Técnica que desactiva aleatoriamente unidades de entrada durante o treinamento, ajudando a evitar sobreajuste.
